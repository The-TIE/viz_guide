# Contributing

## Guide Changes

The visualization guide (`guide/*.md`) is the core knowledge base.
Changes should be made via pull request.

### When to Update the Guide

- Agent consistently makes the same mistake
- New pattern discovered during testing
- Existing guidance is unclear or contradictory

### PR Process

1. Branch from main
2. Edit guide files or apply suggestions from the app
3. Test your changes by generating new visualizations
4. Submit PR with:
   - What issue this fixes
   - Example of before/after behavior

## Code Changes

For app code changes, follow standard PR process.
Run the app locally to test before submitting.
